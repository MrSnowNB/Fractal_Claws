## ticket_io.py Issue Found

Pre-migration ticket_io.load_ticket() built the Ticket using only core fields
and stashed runner extras (task, depends_on, context_files, result_path, etc.)
in a _extras side-dict via object.__setattr__. This meant ticket.task was None
even after loading a YAML with task: set.

Fix: ticket_io.load_ticket() now delegates to Ticket.from_dict() which maps
all declared dataclass fields directly. Only true legacy extras (updated_at,
attempts_log, allowed_tools, max_depth) are preserved in _extras for YAML
round-trip fidelity.
