## Notes

### Fields Not Declared on Ticket Dataclass (src/operator_v7.py)

The following fields are written into YAML but may not be declared on the `Ticket` dataclass:

- `ticket_id` (YAML key) → `id` or `ticket_id` on dataclass
- `depends_on: List[str]`
- `context_files: List[str]`
- `result_path: Optional[str]`
- `task: Optional[str]`
- `depth: int`
- `max_depth: int`
- `status: str`
- `created_at: str`
- `updated_at: str`
- `rationale: str`
- `produces: List[str]`
- `consumes: List[str]`
- `tags: List[str]`
- `agent: str`

**Action required before migration:** Verify these fields exist on `Ticket` with safe defaults.

### as_dict() Shim

The `as_dict()` shim in `src/ticket_io.py` exists for serialization back to YAML.
Do not remove it from `ticket_io.py`. Remove only its usage inside runner.py
for **read access**. Serialization (save_ticket, write YAML) may still use it.

### Deadlock Detection

The dependency graph walk reads `ticket.id` (or `ticket_id`) to match against `depends_on` lists.
Confirm the field name survives migration. If YAML uses `ticket_id` but dataclass uses `id`,
`from_dict()` must map `ticket_id` → `id`.
