## KI-001 — `test_run_command_blocked` fails on Windows

**File:** `tests/test_tools.py`  
**Test:** `test_run_command_blocked_unix`  
**Platform:** Windows (win32)  
**First seen:** STEP-03 baseline gate  

### Root Cause
The original test used `bash -c "rm -rf /"` to verify the blocklist fires.
`bash` is not installed by default on Windows, so the test fails at the
subprocess level before the blocklist even activates.

### Fix Applied
- Renamed to `test_run_command_blocked_unix` with `@pytest.mark.skipif(sys.platform == "win32")`
- Added `test_run_command_blocked_windows` using `cmd /c del /f /s /q c:\\`
- Added matching Windows patterns to `src/tools/terminal.py` blocklist

### Resolution Status
✅ Fixed — cross-platform coverage restored. Unix test skips cleanly on Windows.

## KI-002 — `output/audit.json` and `output/harness_result.json` missing on clean checkout

**File:** `tests/test_harness_artifacts.py`  
**Tests:** `test_runtime_artifact_exists[output/audit.json]`, `test_runtime_artifact_exists[output/harness_result.json]`  
**Platform:** All  
**First seen:** STEP-03 → pre-STEP-04 baseline gate  

### Root Cause
These files are **runtime outputs** generated by `agent/runner.py` executing
TASK-011 (the audit ticket). They are NOT committed to the repo. On a clean
checkout, `git clone`, or after `git clean -fdx`, they do not exist.

The original `test_artifact_exists` parametrized test mixed these runtime
artifacts with static committed artifacts (`output/word_freq.py`,
`output/report.md`), causing 2 failures on every clean run.

### Fix Applied
- Split `ARTIFACTS` into `ARTIFACTS_STATIC` (always enforced) and
  `ARTIFACTS_RUNTIME` (skipped unless `HARNESS_RUN=1` env var is set)
- Runtime artifact tests use `pytest.skip()` when file absent and
  `HARNESS_RUN != 1`
- CI / full harness validation: set `HARNESS_RUN=1` after running
  `python agent/runner.py --no-prewarm`

### Resolution Status
✅ Fixed — default `pytest tests/ -v` run is now clean.
  Full enforcement available via `HARNESS_RUN=1 pytest tests/test_harness_artifacts.py`.
